## Global Rules

1. **Money:** NEVER use `float` or `number` for monetary amounts. Use `Decimal(19,4)` in Prisma, `string` in APIs, `bankersRound()` / `multiply()` / `subtract()` / `add()` from `@lons/common` for all math.

2. **Multi-tenancy:** Every database query must be tenant-scoped. Every service method receives `tenantId` as first argument. No cross-tenant data access.

3. **Idempotency:** Every mutation accepts `idempotencyKey`. Check before executing. Return existing result if key is found.

4. **Events:** Every state transition emits an event via `EventBusService`. All events include `tenantId`, `correlationId`, and a timestamp.

5. **Soft deletes:** No hard deletes for business data. Use `deletedAt`.

6. **Ledger entries:** Append-only. No updates, no deletes. Double-entry bookkeeping.

7. **i18n:** All user-facing strings in the admin portal use `t()` from `useI18n()`. Add keys to `en.json` under appropriate namespaces. Never hardcode strings.

8. **Error responses:** Use structured format `{ code, message, details }`. Import error classes from `@lons/common` (`NotFoundError`, `ValidationError`, etc.).

9. **Test coverage:** 80%+ for all new services. Financial calculation tests are mandatory, not optional.

10. **Naming:** Follow CLAUDE.md naming conventions. Files: kebab-case. Classes: PascalCase. DB tables: snake_case plural. Events: dot.separated.
